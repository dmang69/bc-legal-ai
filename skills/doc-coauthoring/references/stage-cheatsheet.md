# Doc Co-Authoring — Stage Cheatsheet

## Offer

```
I can run a three-stage co-author workflow:
1) Context dump + clarifying Qs
2) Section-by-section brainstorm → curate → draft → edit
3) Reader Testing with a fresh Claude (no context)
This helps the doc work when others (or Claude) read it cold.
Structured workflow, or freeform?
```

## Stage 1 exit

Can discuss edge cases/trade-offs without re-explaining basics.

## Stage 2 section loop

```
Announce section
→ 5–10 clarifying Qs
→ 5–20 brainstorm items
→ Keep/remove/combine
→ Gap check
→ Draft (surgical replace)
→ Edit until done
→ After 3 light passes: what can we cut?
```

## Stage 3 pass criteria

Fresh Claude answers predicted questions correctly; no new ambiguity.

## Anti-patterns

- Brainstorm lists inside the artifact (keep in chat)
- Reprinting whole doc on every edit
- Skipping Reader Testing when audience is non-authors
- Forcing Socratic teaching when user wants a product doc (use this skill, not learning-mode)
