# Failure Mode Catalog — Expanded

The ten failure modes from **Module 3** (`cognitive-awareness`), with symptoms, examples, and antidotes.

**Meta-note:** Modes are not exclusive. One bad response can hit three or four. The check is not “did I hit any?” — it’s “**how many**, and **which is most load-bearing to fix**?”

---

## 1. Confirmation Bias

| | |
|--|--|
| **Symptom** | Reasoning only from evidence that supports the first hypothesis |
| **Example** | User describes a bad-faith eviction; only bad-faith indicators are checked, missing that the **notice is procedurally defective** — often the stronger ground |
| **Antidote** | After forming a hypothesis, actively search for evidence that would **disprove** it. If none exists, hypothesis is stronger. If some exists, weight it before committing |

**Legal/RTB cue:** Always run *procedure first* (form, service, deadline) before deep bad-faith theory.

---

## 2. Anchoring

| | |
|--|--|
| **Symptom** | Sticking to the user’s framing even when it’s wrong |
| **Example** | User asks “how do I win this case?” — assumes winnable. Correct move may reframe: “strongest position may be settlement; here’s why” |
| **Antidote** | Before answering, restate the question in your own terms. If restatement differs meaningfully, address framing before answering |

**Legal/RTB cue:** “How do I win JR?” → first: standing, record, limitation, standard of review — not victory speech.

---

## 3. Availability

| | |
|--|--|
| **Symptom** | Over-weighting the most memorable example rather than the most representative |
| **Example** | Recommending a famous Python library when a less famous one fits better |
| **Antidote** | Ask: “Is this the **right** example, or just the **first** that came to mind?” |

**Legal/RTB cue:** One famous renoviction story ≠ base rate for *this* landlord/unit. Prefer *this* record + verified CanLII patterns if used carefully.

---

## 4. False Precision

| | |
|--|--|
| **Symptom** | Specific number implies confidence you don’t have |
| **Example** | “Success rate is 73%” when basis is “somewhere between half and three-quarters” |
| **Antidote** | No source for the number → use a range or qualitative descriptor. “A minority” beats invented “27%” |

**Legal/RTB cue:** Don’t invent abatement percentages. Ranges + factors + evidence (habitability severity).

---

## 5. Category Error

| | |
|--|--|
| **Symptom** | Treating a situation as one type when it’s another |
| **Example** | Treating a notice as s. 49 (landlord’s use) when the real issue is s. 47 (cause) — different tests, remedies, deadlines |
| **Antidote** | Before applying a framework, verify the category: “Is this actually the right box?” |

**Legal/RTB cue:** Confirm notice form + stated ground against official RTA Part 4 section map (BC Laws). Wrong box = wrong defence.

**Note:** Section numbers must come from current BC Laws — do not rely on memory.

---

## 6. Missing Base Rate

| | |
|--|--|
| **Symptom** | Ignoring how common something actually is |
| **Example** | “Bad-faith evictions are widespread” as fact without rate/evidence |
| **Antidote** | Frequency claim → “What would the base rate have to be?” If unestimable, hedge or research |

**Legal/RTB cue:** Pattern evidence about a *specific* landlord needs *this* history (verified), not “everyone knows.”

---

## 7. Sunk Cost Creep

| | |
|--|--|
| **Symptom** | Defending earlier answer instead of updating on new info |
| **Example** | User adds material facts mid-chat; response rationalizes original rec instead of revising |
| **Antidote** | On new material info: “If I started fresh with all information now available, same answer?” If no → revise. **Say so.** |

**Legal/RTB cue:** New service proof or payment ledger can flip “non-payment” defence. Update immediately; handoff to self-improvement if the skill map was wrong.

---

## 8. Solution-First

| | |
|--|--|
| **Symptom** | Pick answer, reason backward to justify it |
| **Example** | Decide “client should settle” before analyzing the case, then build settlement case only |
| **Antidote** | Analysis first. Recommendation last. If preferred answer no longer fits analysis, analysis wins |

**Legal/RTB cue:** IRAC per issue → remedy. Settlement is an outcome of leverage math, not a default mood.

---

## 9. Overclaiming

| | |
|--|--|
| **Symptom** | Preference or judgment stated as fact |
| **Example** | “Python is better than JavaScript” without use-case |
| **Antidote** | “Would a knowledgeable person who disagreed have a legitimate position?” If yes, you’re overclaiming — scope the claim |

**Legal/RTB cue:** “You will win” → overclaim. “On these facts, grounds A/B are stronger because…” → calibrated (Inferred).

---

## 10. Under-Decomposition

| | |
|--|--|
| **Symptom** | Complex question collapsed into one answer |
| **Example** | “Should I take this deal?” = price, counterparty, fallback, opportunity cost, timing — one word misses all five |
| **Antidote** | Multi-sentence question → decompose into sub-questions and answer each |

**Legal/RTB cue:** “Should I dispute?” → deadline, ground, evidence, cost, stay risk, settlement leverage — not a bare yes/no.

## Steel-man pass (reminder)

For judgment/recommendation: state position → strongest opposing position → does it change the rec? → revise or keep (more robust for having checked). Surface only when material.
