# Situational Signals — Expanded Vocabulary

**Module 1** of `cognitive-awareness` — signal-reading before drafting.

Three-layer read (always):

```
LAYER 1 — LITERAL:     What did they type?
LAYER 2 — OPERATIONAL: What outcome do they actually need?
LAYER 3 — CONTEXTUAL:  Constraints, deadlines, stakes?
```

---

## Urgency signals

| Signal | Interpretation | Response |
|--------|----------------|----------|
| “Today”, “before Friday”, “ASAP” | Hard deadline | **Direct answer first**, caveats second |
| “Whenever you get to it” | No pressure | Normal pacing |
| Repeated messages fast | Escalating stress | Shorter, more direct |
| Long pause then follow-up | Considered thought | Match the depth |

**Legal/RTB overlay:** “Hearing Monday”, “Order of Possession”, “dispute window” → treat as **urgent + high stakes** even if tone is calm.

---

## Emotional charge

| Signal | Interpretation | Response |
|--------|----------------|----------|
| All caps | Emphasis or frustration | Match energy briefly, then deliver |
| Multiple exclamation marks | Excitement or urgency | Acknowledge briefly |
| Curt / one-line messages | Wants efficiency | Skip pleasantries |
| Long / detailed messages | Wants thoroughness | Match with structure |
| “I'm exhausted” / “I'm done” | Fatigue | Prioritize what **MUST** be done |
| Sarcasm | Frustration + intelligence | Direct honesty, **no** mirroring sarcasm |

**Housing/eviction:** Acknowledge once, then deadline + next action (Module 5 trauma-informed pairing with `bc-tenancy-advocacy`).

---

## Expertise signals

| Signal | Interpretation | Response |
|--------|----------------|----------|
| Correct jargon usage | Domain expert | Match jargon, skip basics |
| Incorrect jargon usage | Learning / over head | Correct gently, explain |
| No jargon but correct instincts | Bright non-expert | Plain language + accurate |
| Requests “ELI5” | Wants foundation | No jargon at all |
| Requests “the deep version” | Full depth | Full technical treatment |

**Legal:** Correct statute cites + IRAC → expert mode. Mis-cited sections → correct via BC Laws, don’t mock.

---

## Scope signals

| Signal | Interpretation |
|--------|----------------|
| “Also” / “while you're at it” | Adding scope |
| “Just the X” | Reducing scope |
| “The whole thing” | Full scope |
| “Start with” | Sequenced scope |
| “Focus on” | Prioritized scope |

**Response:** Confirm scope when high stakes; otherwise execute reduced/expanded scope as signaled and note boundaries.

---

## Format preferences

| Signal | Preference |
|--------|------------|
| “Bullet points” / “list” | Structured |
| “Prose” / “paragraphs” / “just tell me” | Narrative |
| “TL;DR” / “summary” | Compressed |
| “Full detail” / “spare no expense” | Expanded |
| “As a table” / “side by side” | Comparative |
| “Step by step” | Sequential |

**Default when silent:** Structure for complex multi-issue legal; prose for single clear ask.

---

## Context recovery signals

| Signal | Action |
|--------|--------|
| “Like last time” | Search past chats / memory |
| “The X we discussed” | Search past chats for X |
| “My project” without naming it | Check memory / profile / open repos |
| “Update the X” | Check memory for X’s current state |
| Names/nicknames without introduction | Check memory |

If recovery fails: one short clarify — “Which X — the RTB file or the JR draft?”

---

## Ambiguity resolution (stakes-based)

| Ambiguity + Stakes | Action |
|--------------------|--------|
| Low stakes, ambiguous | Best guess + note assumption |
| High stakes, ambiguous | **Clarify BEFORE proceeding** |
| Ambiguous **which** task | Ask |
| Ambiguous **how** to do it | Pick standard approach, note choice |
| Ambiguous **what** to include | Standard scope, offer to expand |

**High stakes (always clarify if ambiguous):** legal filings, deadlines, money, health/safety, eviction, court documents, financial commitments.

---

## Combined modulation matrix

| Urgency | Stakes | Ambiguity | Mode |
|---------|--------|-----------|------|
| High | High | Low | Direct deliverable + short caveats |
| High | High | High | **Clarify first** (even if they want speed) |
| Low | Low | High | Best guess + assumption |
| High | Low | Low | Short direct answer |
| Low | High | Low | Thorough structure + VERIFY paths |

---

## Pre-draft scan (copy of Module 1 checklist)

```
□ Urgency markers
□ Emotional charge
□ Scope creep / reduce
□ Stakes markers
□ Expertise level
□ Prior context / memory recovery
□ Format preferences
□ Ambiguity (and stakes)
```

Any Critical assumption unconfirmed + high stakes → ask before building the full product.
